# API Structure для бэкенда

## Структура продукта (Product)

Каждый продукт должен содержать следующие поля:

```json
{
  "id": 1,
  "name": "Поплин",
  "name2": "зима/лето",
  "price": "37 800 ₽",
  "material": "Поплин",
  "color": "Тёмный",
  "article": "KL–00032541",
  "description": "наволочка 5 × 0 см — 2 шт, пододеяльник 20 × 20 см, простыня 24 × 70 см. Рекомендации по уходу: стирка при температуре до 30°С; не отбеливать; гладить при средней температуре (до 150°С); только сухая химчистка; разрешена барабанная сушка.",
  "visit": ["1,5 сп", "2 сп", "евро", "дует"],
  "sizes": ["1,5 сп", "2 сп -nh", "евро", "дует -nh"],
  "images": ["/about-img.png", "/about-img.png"],
  "img": "/card-img.png",
  "have": 4
}
```

### Описание полей:

- `id` (integer) - уникальный идентификатор продукта
- `name` (string) - основное название продукта
- `name2` (string) - дополнительное название (может быть пустой строкой "")
- `price` (string) - цена с символом рубля
- `material` (string) - материал полотна (Поплин, Сатин, Бязь, Хлопок, Лён, Трикотаж)
- `color` (string) - цвет товара
- `article` (string) - артикул товара
- `description` (string) - полное описание товара
- `visit` (array of strings) - размеры для отображения в карточке товара
  - Размеры с суффиксом `-nh` отображаются серым цветом (недоступны)
  - Пример для хитов: `["1,5 сп", "2 сп -nh", "евро -nh", "дует -nh"]`
  - Пример для остальных: `["1,5 сп", "2 сп", "евро", "дует"]`
- `sizes` (array of strings) - размеры для страницы детального просмотра
  - Размеры с суффиксом `-nh` отображаются серым и неактивны
  - **ВАЖНО**: Суффикс `-nh` используется ТОЛЬКО для товаров с id 7, 8, 9 (блок "Хиты продаж")
  - Пример для хитов: `["1,5 сп", "2 сп -nh", "евро -nh", "дует -nh"]`
  - Пример для остальных: `["1,5 сп", "2 сп", "евро", "дует"]`
- `images` (array of strings) - массив URL изображений для галереи на странице товара
- `img` (string) - основное изображение для карточки товара
- `have` (integer) - количество товара в наличии (для блока "Хиты продаж")

### Важные замечания:

1. **Размеры постельного белья** (id 1-17): используются размеры "1,5 сп", "2 сп", "евро", "дует"
2. **Размеры одежды** (id 18-24 трикотаж): используются размеры одежды:
   - Халаты (id 18): `["40", "42", "44", "46"]`
   - Сорочки (id 19): `["42", "44", "46", "48"]`
   - Пижамы (id 20): `["44", "46", "48", "50"]`
   - Туники (id 21): `["40", "42", "44", "46"]`
   - Толстовки (id 22): `["S", "M", "L", "XL"]`
   - Спортивные костюмы (id 23): `["M", "L", "XL", "XXL"]`
   - Трикотаж большая карточка (id 24): `["40", "42", "44", "46", "48", "50"]`
3. **Суффикс -nh**: означает "not have" (нет в наличии), отображается серым цветом
   - **КРИТИЧЕСКИ ВАЖНО**: используется ТОЛЬКО для товаров с id 7, 8, 9 (блок "Хиты продаж")
   - Для всех остальных товаров размеры БЕЗ суффикса `-nh`
4. **name2**: может быть пустой строкой, если нет дополнительного названия
5. **URL структура**: `/about/{name}--{name2}--{id}` где пробелы заменяются на `-`, а части разделяются `--`
   - Пример: `/about/Поплин--зима-лето--1`
   - Если в name2 есть `/`, то заменяется на `-` в URL

## Необходимые эндпоинты:

### 1. Получить все продукты

```
GET /Products/
Response: Array of Product objects
```

### 2. Получить один продукт по ID

```
GET /Products/{id}/
Response: Product object
```

### 3. Фильтрация по материалу (опционально)

```
GET /Products/?material=Поплин
Response: Array of Product objects
```

## Тестовые данные

Всего нужно 24 продукта:

- id 1-6: для страницы "Распродажа"
- id 7-9: для блока "Хиты продаж" (с полем have)
- id 10-12: для каталога (Бязь, Хлопок, Лён)
- id 13-17: для блока "Новинки" (5 продуктов)
- id 18-24: для блока "Трикотаж" (7 продуктов)

Полный список тестовых данных находится в файле `src/data/products.js`
